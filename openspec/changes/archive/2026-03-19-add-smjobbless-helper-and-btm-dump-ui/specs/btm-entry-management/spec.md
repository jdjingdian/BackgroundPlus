## 新增需求

### 需求:设置入口可达
系统必须提供可达的设置入口，使用户可以进入 helper 安装与状态管理页面。

#### 场景:用户打开设置页
- **当** 用户在应用中进入设置
- **那么** 系统必须展示 helper 安装引导与当前状态信息

## 修改需求

### 需求:BTM 条目可视化
系统必须提供 BTM 条目列表与详情展示，至少包含 identifier、type、url、generation、parent-child 关系信息；当 helper 已安装时，数据源必须来自真实系统 dump 结果而非固定占位数据。

#### 场景:成功展示条目
- **当** 系统可通过 helper 获取 dumpbtm 输出
- **那么** 用户可以看到结构化条目与关系信息

#### 场景:解析不完整
- **当** dumpbtm 输出存在缺字段或格式漂移
- **那么** 系统必须提示“解析不完整”并继续展示可用条目

#### 场景:helper 未安装
- **当** 用户打开列表页且 helper 尚未安装
- **那么** 系统必须展示“需先安装 helper”的状态提示和前往设置的引导
