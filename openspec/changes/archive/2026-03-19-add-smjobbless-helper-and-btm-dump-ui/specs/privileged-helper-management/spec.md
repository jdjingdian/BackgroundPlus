## 新增需求

### 需求:Helper Target 与身份约束
系统必须提供独立的特权 helper target，且其包标识必须为 `cn.magicdian.BackgroundPlus.helper`，用于承载 BTM 相关特权任务。

#### 场景:构建产物包含 helper target
- **当** 开发者执行工程构建
- **那么** 构建产物中必须包含 `cn.magicdian.BackgroundPlus.helper` 对应 target 且可被主程序引用

### 需求:SMJobBless 安装引导
系统必须在设置界面提供 helper 安装引导，并明确展示未安装、安装中、已安装、安装失败四类状态。

#### 场景:用户主动安装 helper
- **当** helper 状态为未安装且用户在设置界面触发安装
- **那么** 系统必须发起 SMJobBless 安装流程并在界面实时更新安装状态

#### 场景:安装失败反馈
- **当** SMJobBless 返回失败
- **那么** 系统必须展示可读的失败信息和重试入口

### 需求:受控特权调用边界
系统必须通过 helper 暴露受控接口执行特权任务，主程序禁止绕过 helper 直接执行 BTM 特权修改。

#### 场景:请求读取 BTM dump
- **当** 主程序请求读取 BTM dump 数据
- **那么** 请求必须通过 helper 接口完成并返回结果给主程序

#### 场景:主程序尝试直连特权写操作
- **当** 主程序代码路径尝试直接执行特权写操作
- **那么** 系统必须拒绝该路径并要求通过 helper 受控接口执行
