## 为什么

当前 Helper 安装界面没有可见的关闭按钮，用户只能通过 `Esc` 退出，交互不直观且可发现性差。同时“关于”页面与“通用”页面在视觉层级和信息组织上不够统一，整体观感较弱，影响产品易用性与专业感。

## 变更内容

- 为 Helper 安装界面增加明确的关闭入口，并保留键盘快捷退出能力。
- 重构并统一“通用”与“关于”界面的布局、间距、状态展示和操作区样式，参照提供的设计方向提升美观度。
- 将 Helper 安装状态、安装/卸载操作与说明文案整合到统一信息层中，减少理解成本。
- 优化“关于”界面的应用信息展示（应用图标、名称、版本、描述、许可证与项目入口），与主设置页保持一致的视觉语言。

## 功能 (Capabilities)

### 新增功能
- `helper-install-and-about-ui`: 统一定义 Helper 安装页与关于页的交互和视觉规范，包括关闭行为、状态呈现与信息布局。

## 影响

- 受影响代码：设置窗口相关 UI 组件、Helper 状态管理与安装操作触发逻辑、关于页展示组件。
- 受影响系统：桌面端设置/偏好窗口交互流程。
- 对外 API：无新增外部 API；主要为内部界面行为与展示层改动。
