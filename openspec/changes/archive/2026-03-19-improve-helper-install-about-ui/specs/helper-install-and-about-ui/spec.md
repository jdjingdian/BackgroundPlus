## ADDED Requirements

### 需求:Helper 安装页必须提供可见关闭入口
系统必须在 Helper 安装页提供明确的可点击关闭按钮，且禁止仅依赖键盘快捷键作为唯一退出方式。

#### 场景:用户通过关闭按钮退出
- **当** 用户位于 Helper 安装页并点击关闭按钮
- **那么** 系统关闭当前设置窗口或退出该安装界面流程

#### 场景:用户通过 Esc 退出
- **当** 用户位于 Helper 安装页并按下 `Esc`
- **那么** 系统执行与关闭按钮一致的退出行为

### 需求:Helper 状态与操作必须在同一信息层展示
系统必须在“通用”页以统一信息块展示 Helper 当前状态、状态说明与对应操作按钮，并根据状态切换按钮可用性和文案。

#### 场景:已安装状态
- **当** Helper 已安装且可用
- **那么** 系统显示“已安装”状态标识，并提供可执行的卸载操作

#### 场景:未安装状态
- **当** Helper 未安装或不可用
- **那么** 系统显示“未安装”或等价状态标识，并提供可执行的安装操作

### 需求:关于页必须展示统一应用信息结构
系统必须在“关于”页展示应用图标、应用名称、版本信息、简要说明、许可证声明与项目主页入口，并与“通用”页保持一致的布局节奏和视觉层级。

#### 场景:查看关于页核心信息
- **当** 用户打开“关于”页
- **那么** 用户可以在单页内看到应用图标、名称、版本和简要说明

#### 场景:访问项目主页
- **当** 用户点击“项目主页”按钮
- **那么** 系统打开项目主页链接
