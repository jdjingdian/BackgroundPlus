## 修改需求

### 需求:BTM 条目可视化

系统 MUST 提供以“侧栏功能分组 + 主区域列表”为结构的 BTM 条目展示。侧栏必须至少包含“登录时打开”和“允许在后台”两个入口；主区域列表必须仅展示当前入口对应的条目集合，并包含每行的 icon、主标题、副标题、开关状态与行级 `→` 操作入口。当 helper 已安装时，数据源必须来自真实系统 dump 结果而非固定占位数据。

#### 场景:进入登录时打开分组
- **当** 用户在侧栏选择“登录时打开”
- **那么** 系统必须只展示被归类为登录时打开的条目，并隐藏允许在后台条目

#### 场景:进入允许在后台分组
- **当** 用户在侧栏选择“允许在后台”
- **那么** 系统必须只展示被归类为允许在后台的条目，并隐藏登录时打开条目

#### 场景:分组空状态独立展示
- **给定** 当前分组没有可展示条目
- **当** 用户进入该分组
- **那么** 系统必须展示该分组的空状态提示，且不得影响另一个分组的可访问性

#### 场景:分类不完整时可用优先
- **给定** dumpbtm 输出存在缺字段或格式漂移，导致部分条目分类不确定
- **当** 系统完成解析与分类
- **那么** 系统必须提示“分类不完整”并继续展示可确定分类的条目

#### 场景:helper 未安装
- **当** 用户打开任一 BTM 分组且 helper 尚未安装
- **那么** 系统必须展示“需先安装 helper”的状态提示和前往设置的引导

#### 场景:行内开关与跳转互不干扰
- **给定** 用户位于某条目行
- **当** 用户点击开关或点击 `→` 按钮
- **那么** 系统必须仅触发对应操作，且不得误触发另一个控件行为

#### 场景:进入详情保持上下文
- **给定** 用户点击某行的 `→` 按钮
- **当** `BackgroundItemDetailView` 显示
- **那么** 系统必须以从右侧滑入的动画呈现详情，并让对应分组列表继续维持当前滚动与选中状态

#### 场景:开关必须是 switch 样式
- **当** 用户查看任意条目行中的开关控件
- **那么** 该控件必须呈现为 macOS switch 风格，且 `accessibilityLabel` 与 `accessibilityValue` 表示当前状态
