## 为什么

当前 `sfltool dumpbtm` 的结果在 UI 中被混合展示，用户无法快速区分“登录时打开（Login Items）”与“允许在后台（Background Items）”。这会让定位条目来源和排查行为变得困难，尤其在存在同名 helper 或 legacy daemon 时更明显。

需要将同一份 dump 数据按条目语义拆分为两个独立视图分组，并在左侧侧栏提供明确入口，以匹配系统设置的认知模型。

## 变更内容

- 将 BTM 列表从单一入口改为两个侧栏入口：`登录时打开` 与 `允许在后台`。
- 在解析层新增“条目分类投影”，把同源 dump 结果映射到两个集合，并保持原始条目可追踪。
- 每个分组仅展示对应类别条目，支持保持原有行能力（图标、标题、副标题、开关与进入详情）。
- 增加空状态与错误提示语义：某分组无数据时仅影响当前分组展示，不影响另一个分组。
- 保持既有 helper 未安装、解析不完整、详情跳转与开关交互语义不变。

## 功能 (Capabilities)

### 新增功能

- 无

### 修改功能

- `btm-entry-management`: 将“后台模块”单列表展示能力扩展为“按登录时打开/允许在后台分组展示”，并定义分类、筛选与分组空状态行为。

## 影响

- 受影响模块：BTM 解析映射层、侧栏路由与列表容器视图。
- 受影响行为：列表入口信息架构、分组过滤逻辑、空状态文案分流。
- 测试影响：需要新增/调整分类映射测试、分组渲染测试和回归测试（开关与详情跳转不回归）。
