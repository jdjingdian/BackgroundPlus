## 上下文

当前实现已具备 helper 安装、能力读取和基础状态展示，但存在三个缺口：
1. helper 缺少受控自卸载链路，安装后回收依赖手工清理，失败时缺少一致反馈。
2. 设置页中“卸载”与常规操作视觉层级接近，风险语义不足。
3. 虽已将 FDA 纳入 helper 能力判定，但设置界面缺少持续监听与一键修复入口，用户在未授权时需要自行查找系统设置路径。

变更将跨越 helper（XPC/卸载执行）、App 状态层（权限检测与监听）、设置 UI（Tab 与动作样式）三个模块。

## 目标 / 非目标

**目标：**
- 建立可重复执行、可观测的 helper 自卸载流程，并与现有 helper 生命周期管理保持一致。
- 在设置 UI 中强化高风险操作语义，将卸载入口定义为红色危险样式。
- 在设置界面增加 FDA 权限管理入口，实时显示权限状态并在未开启时提供系统设置跳转。
- 保证权限状态、helper 能力状态与可执行操作三者一致，避免“按钮可点但操作必失败”。

**非目标：**
- 不引入新的系统私有权限或绕过 FDA 的技术路径。
- 不在本次变更中重构全部设置页导航，仅新增必要 Tab/区块。
- 不扩展与本需求无关的 helper 写操作能力。

## 决策

1. 决策：沿用现有 app -> helper 的受控调用边界实现自卸载，不允许 app 直接执行底层卸载命令。
- 原因：保持特权操作单入口，便于审计、错误映射和版本兼容控制。
- 备选方案：app 直接执行脚本或 shell 卸载。已否决，安全边界与可观测性变差。

2. 决策：将自卸载设计为“请求确认 -> 执行卸载 -> 主程序刷新状态”的三段式流程。
- 原因：避免误触，且可在完成后立即回收 UI 状态（已卸载/失败原因）。
- 备选方案：单击即刻卸载且不刷新。已否决，误操作风险高且状态易失真。

3. 决策：在设置中引入“权限管理”Tab（或同级独立区块），集中展示 FDA 状态并提供深链跳转。
- 原因：将权限诊断与修复动作前置，减少用户在系统设置中盲找路径。
- 备选方案：仅在错误弹窗中提供说明文本。已否决，触达晚且恢复路径长。

4. 决策：FDA 状态采用“主动检测 + 前台刷新”机制，而非仅在进入页面时单次判定。
- 原因：用户在系统设置切换权限后可立即看到状态变化，减少重启或重复进入成本。
- 备选方案：单次检测。已否决，体验不连续且易产生误判。

5. 决策：卸载按钮使用统一危险样式（红色背景/危险语义文案），并仅在可卸载状态可点击。
- 原因：符合 macOS 风险操作心智模型，降低误触概率。
- 备选方案：保留默认样式。已否决，风险提示不足。

## 风险 / 权衡

- [风险] 不同 macOS 版本系统设置深链存在差异，可能跳转失败。 → 缓解：实现主路径 + 回退路径（打开系统设置并定位“隐私与安全性”）。
- [风险] 自卸载执行期间 helper 连接中断导致 UI 状态短暂不一致。 → 缓解：执行后强制触发状态重读，并为“未知中间态”提供加载提示。
- [风险] 持续监听 FDA 状态增加轮询或通知复杂度。 → 缓解：优先使用低频前台刷新与关键节点重检，避免高频后台轮询。
- [风险] 卸载入口过于显眼导致误操作担忧。 → 缓解：保留二次确认与结果反馈，不直接静默执行。

## Migration Plan

1. 在 helper 接口层新增/接入自卸载调用，补齐结果码与错误映射。
2. 在 app 状态管理层新增 FDA 状态监听与刷新触发点，并暴露给设置页。
3. 在设置页新增“权限管理”Tab（或等价区块），接入状态展示和系统设置跳转按钮。
4. 调整“卸载”按钮样式为危险样式，接入确认流程与执行态。
5. 联调验证：
   - helper 已安装且 FDA 已开启：可正常卸载并刷新为未安装。
   - helper 已安装但 FDA 未开启：权限管理页显示未开启并可跳转系统设置。
   - 跳转失败场景：给出可读 fallback 提示。

回滚策略：
- 若自卸载链路异常，可临时回退到“仅展示卸载引导、不执行实际调用”的安全模式；
- 若权限深链在特定系统版本不稳定，可降级为打开系统设置通用入口并保留手动指引。
