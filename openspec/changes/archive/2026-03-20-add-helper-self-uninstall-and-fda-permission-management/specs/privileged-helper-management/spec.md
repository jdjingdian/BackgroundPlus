## ADDED Requirements

### 需求:Helper 必须提供受控自卸载能力
系统 MUST 提供由 app 通过 helper 接口触发的受控自卸载流程，且卸载执行必须在 helper 边界内完成，不得允许 app 绕过 helper 直接执行底层卸载路径。

#### 场景:用户在已安装状态触发卸载
- **当** helper 已安装且用户在设置界面确认执行卸载
- **那么** 系统必须通过 helper 接口执行自卸载，并在完成后返回“成功卸载”结果给 app

#### 场景:卸载执行失败
- **当** helper 自卸载过程中出现权限、连接或系统调用失败
- **那么** 系统必须返回可读错误并保持状态为“仍已安装/卸载失败”，禁止静默标记为已卸载

### 需求:自卸载结果必须与能力状态同步刷新
系统 MUST 在自卸载操作完成后立即刷新 helper 安装状态与能力状态，避免 UI 持续显示过期状态。

#### 场景:卸载成功后的状态回收
- **当** helper 返回卸载成功
- **那么** 系统必须将 helper 状态更新为未安装，并禁用依赖 helper 的特权操作入口

#### 场景:卸载失败后的状态保持
- **当** helper 返回卸载失败
- **那么** 系统必须保持 helper 状态为已安装并展示失败原因与重试入口
