## 为什么

当前界面将后台模块混在侧边栏中，信息密度高且切换语义不清晰，用户很难区分“功能导航”和“模块数据列表”。
用户希望交互更接近系统原生设置：侧边栏用于功能切换，主区域用于列表化展示后台模块，并可从每一行快速进入定制详情。

## 变更内容

- 将侧边栏重构为“功能导航区”，不再直接承载完整后台模块列表。
- 在主区域新增后台模块列表视图，按系统设置风格展示名称、副标题、图标、状态开关。
- 在每行开关右侧新增“→”操作按钮，用于进入“我的定制功能”详情页。
- “我的定制功能”页面复用当前应用主页面已有的每个 app 详情信息能力，并从列表行内可直达。
- 统一列表行视觉结构与间距，优先使用系统原生控件与图标风格（例如 NSImage/系统 icon 渲染）。

## 功能 (Capabilities)

### 新增功能
- `custom-entry-navigation`: 支持从后台模块列表行内一键跳转到“我的定制功能”详情页。

### 修改功能
- `btm-entry-management`: 调整信息架构与主界面布局，改为“侧栏功能切换 + 主区模块列表”，并在列表中展示模块 icon 与行级操作。

## 影响

- 受影响模块：主窗口导航容器、BTM 列表视图、行组件、详情页路由/状态同步。
- 受影响行为：列表选中逻辑、开关交互区域、详情页进入路径。
- 兼容性影响：不改变底层 BTM 数据来源与开关能力，主要为 UI 信息架构与交互路径变更。
