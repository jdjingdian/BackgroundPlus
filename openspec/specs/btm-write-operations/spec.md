# btm-write-operations 规范

## 目的
待定 - 由归档变更 btm-single-delete-toggle-write-path 创建。归档后请更新目的。
## 需求
### 需求:Helper 写能力协商
系统必须在执行任何 BTM 写操作前完成 helper 写能力协商；当 helper 未声明支持写能力时，系统必须阻断写操作并降级为只读。

#### 场景:写能力可用
- **当** app 读取到 helper capabilities 且 `supportsWriteOperations=true`
- **那么** 系统必须允许进入开关与删除写入流程

#### 场景:写能力不可用
- **当** app 读取到 helper capabilities 且 `supportsWriteOperations=false`
- **那么** 系统必须阻断所有 BTM 写操作并提示“当前环境仅支持只读”

### 需求:单条开关写操作
系统必须支持针对单个 BTM 条目执行启用/禁用写操作，并在写入成功后刷新真实 dump 数据。

#### 场景:开关写入成功
- **当** 用户切换某条目开关且 helper 写操作返回成功
- **那么** 系统必须刷新条目列表并展示与真实系统状态一致的开关值

#### 场景:开关写入失败
- **当** 用户切换某条目开关且 helper 返回错误
- **那么** 系统必须回滚本地开关显示并展示错误信息

### 需求:单点删除真实执行
系统必须支持针对单个目标条目执行删除写操作，并保持写前备份与写后校验语义。

#### 场景:删除写入成功
- **当** 用户完成风险确认并触发删除，且 helper 执行成功
- **那么** 系统必须返回成功结果、展示备份路径并在刷新后不再显示目标条目

#### 场景:删除写入失败
- **当** 用户触发删除且 helper 返回执行失败
- **那么** 系统必须展示失败结果并保留可追溯错误信息
