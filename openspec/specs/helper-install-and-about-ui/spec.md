# helper-install-and-about-ui 规范

## 目的
定义 Helper 安装页与关于页的信息结构、退出交互和状态操作一致性约束，确保设置流程清晰、可见且可执行。
## 需求
### Requirement: Helper 安装页必须提供可见关闭入口

系统必须在 Helper 安装页提供明确的可点击关闭按钮，且禁止仅依赖键盘快捷键作为唯一退出方式。

#### Scenario: 用户通过关闭按钮退出

- **WHEN** 用户位于 Helper 安装页并点击关闭按钮
- **THEN** 系统关闭当前设置窗口或退出该安装界面流程

#### Scenario: 用户通过 Esc 退出

- **WHEN** 用户位于 Helper 安装页并按下 `Esc`
- **THEN** 系统执行与关闭按钮一致的退出行为

### 需求:Helper 状态与操作必须在同一信息层展示
系统必须在“通用”页以统一信息块展示 Helper 当前状态、状态说明与对应操作按钮，并根据状态切换按钮可用性、文案与风险样式；其中卸载操作必须使用红色危险样式以突出高风险语义。

#### 场景:已安装状态
- **当** Helper 已安装且可用
- **那么** 系统显示“已安装”状态标识，并提供可执行的红色“卸载”按钮

#### 场景:未安装状态
- **当** Helper 未安装或不可用
- **那么** 系统显示“未安装”或等价状态标识，并提供可执行的安装操作

### Requirement: 关于页必须展示统一应用信息结构

系统必须在“关于”页展示应用图标、应用名称、版本信息、简要说明、许可证声明与项目主页入口，并与“通用”页保持一致的布局节奏和视觉层级。

#### Scenario: 查看关于页核心信息

- **WHEN** 用户打开“关于”页
- **THEN** 用户可以在单页内看到应用图标、名称、版本和简要说明

#### Scenario: 访问项目主页

- **WHEN** 用户点击“项目主页”按钮
- **THEN** 系统打开项目主页链接

### 需求:设置界面必须提供权限管理视图与 FDA 快速修复入口
系统 MUST 在设置界面提供“权限管理”Tab（或同级独立视图），用于展示完全磁盘访问（FDA）状态，并在未开启时提供可点击的系统设置跳转按钮。

#### 场景:FDA 未开启时显示修复入口
- **当** 系统检测到应用未获得完全磁盘访问权限
- **那么** 权限管理视图必须显示“未开启”状态与“前往系统设置”按钮

#### 场景:用户点击前往系统设置
- **当** 用户在权限管理视图点击“前往系统设置”按钮
- **那么** 系统必须尝试跳转到对应的系统设置页面，若定向失败则至少打开系统设置并给出可读引导

#### 场景:FDA 开启后状态更新
- **当** 用户完成授权并返回应用
- **那么** 权限管理视图必须在刷新后显示“已开启”状态，并隐藏或禁用修复按钮
